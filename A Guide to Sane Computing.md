# BE NORMAL: A Guide to Sane Computing

## 1. USE MAINSTREAM OPERATING SYSTEMS

- macOS or Windows are perfectly fine choices. They work, they're supported, and people know how to help you.
- Linux is great for servers and specific use cases, but for daily desktop use? You're making life harder than it needs to be.
- Haiku, Plan 9, TempleOS? Cool projects, but not for your daily driver. Save the hobby OS experiments for a VM.

## 2. USE THE RIGHT TOOL FOR THE JOB

- If VS Code works for you, use it. You don't need Vim mastery points.
- If a GUI tool gets the job done faster, use it. CLI is not a moral virtue.
- If a library/framework is popular and well-documented, it's probably the right choice.
- Don't reinvent wheels. Don't build your own CMS when WordPress exists... unless you 100% want to, you see the tool as imperfect, or the tool doesn't work for what you have in mind.

## 3. PROPRIETARY TOOLS ARE INNOCENT UNTIL PROVEN GUILTY

- GitHub Copilot, ChatGPT, Claude? If they help you ship faster, use them.
- Figma, Adobe Creative Cloud, JetBrains IDEs? They're industry standards for a reason.
- Open source is great, but "proprietary" is not automatically evil.
- Judge tools by: Does it work? Is it maintained? Can I get help when I'm stuck?

## 4. VIBE CODING > PERFECT ARCHITECTURE

- Ship the thing. Perfect is the enemy of done.
- If it works and users are happy, you did your job.
- Refactor when it hurts, not because you read a blog post about "clean code."
- Your code will probably be rewritten in 2 years anyway. Don't overthink it.

## 5. DON'T BE A PURIST

- It's okay to mix languages. Python for AI, Node.js/PWA/plain HTML for web, bash for scripts.
- It's okay to use a database that isn't your favorite.
- It's okay to copy-paste Stack Overflow answers.
- It's okay to not care about the latest framework hype cycle.

## 6. PRACTICALITY > IDEOLOGY

- The goal is to solve problems, not to make a philosophical statement.
- If a proprietary tool saves you 10 hours a week, that's 10 hours you can spend on things that matter.
- If a mainstream framework has 100x more tutorials than your niche choice, that matters.
- Your users don't care about your tech stack. They care that it works.

## 7. BE KIND TO YOURSELF

- You don't need to know everything.
- It's okay to use tools that do things for you.
- It's okay to not understand how everything works under the hood.
- You're a problem solver, not a martyr.

## 8. THE "NORMAL" CHECKLIST

Before choosing a tool or approach, ask:
- Will this be easy to maintain in 6 months?
- Can I find help if I get stuck?
- Is this a standard choice, or am I being clever?
- Does this actually solve my problem, or am I just having fun?

If you're choosing the harder path for ideological reasons, reconsider.

---

Remember: The best tool is the one that lets you finish your work and go do something else.

Be normal. Ship stuff. Be happy.
